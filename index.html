Idee

Das Projekt ist unterteilt in drei Subziele:
Das erste Ziel ist das Aufsetzen einer Arbeits-/Testumgebung, welche die Anforderungen der fünf Jahrgänge, welche sich in installierter Software unterscheiden, der Informationstechnologie individuell erfüllt, sowie die Möglichkeit des Aufsetzens von Windows 10. Das Deployment beider Varianten soll per Windows Deployment Services (WDS) geschehen.

Das zweite Ziel ist das Erstellen von zwei Übungen, welche den Schülerinnen und Schülern den Stoff des vierten und fünften Jahrgangs näherbringen oder auffrischen soll.

Das dritte Ziel ist das Programmieren einer Software, welche den Soll-Zustand mit dem Ist-Zustand der ausgelesenen Konfiguration anhand von show-Befehlen von Virtual-GNS3 Routern und Switches ausliest, speichert, vergleicht und bewertet. Für den Soll-Zustand kann die Lehrkraft auswählen, welcher Befehl welche Wertung hat, ob er eins-zu-eins, in einem vorgegebenen Bereich liegen muss, oder optional in dem Sollzustand vorhanden sein soll.

Ziele

RE-H 1 Aufsetzten eines WDS-Servers und Deployment von Windows 10

Ein Windows Deployment Services Server ist aufgesetzt, welcher in einem privaten Netz alle PCs, welche per PXE-Boot starten, neu mit Windows 10 installiert.

RE-H 2 Deployment eines Windows Server 2019 VHD-Implementierung

Der WDS-Server hat die Auswahlmöglichkeit nun auch einen Windows Server 2019 Core zu deployen. VHD-Dateien sind ebenfalls erstellt und individuell angepasst, um anschließend auf den PCs, auf welchen Windows Server 2019 Core aufgesetzt wurde, verfügbar zu machen.

RE-H 3 Automatisierung der VHD-Implementierung durch Skripte

Das Mounten und Booten der unterschiedlichen VHD-Dateien auf den PCs, auf welchen Windows Server Core 2019 deployt wurde, ist per Skript implementiert. Die Lehrkraft muss für jeder Computer das richtige Skript für die passende VHD Datei auswählen, alle anderen Vorgänge sind automatisiert.

RE-H 4 Erstellen zweier Übungstopologien

Es sind zwei Topologien, welche den Stoffumfang des vierten und fünften Jahrgangs umfassen, sowie deren Angaben und Musterlösungen, erstellt.

RE-H 5 Autograder – Vergleich von zwei Konfigurationsdateien

Mit Python ist ein Programm geschrieben, welches den Sollzustand mit dem Istzustand zweier Konfigurationsdateien vergleicht und bewertet. Die Konfigurationsdatei ist die Running-Config eines Routers oder Switches.

RE-H 6 Autograder – Auslesen der Konfiguration

Auslesen der Konfiguration unter Verwendung von show-Befehlen. Dazu wird die IP-Adresse des Computers, auf dem GNS3 ausgeführt wird, benötigt, damit anschließend mit Nmap alle offenen Ports evaluiert werden können. Somit hat das Programm alle benötigten Informationen, um Telnet Verbindungen herstellen zu können und Daten des virtuelle Netzwerkgerätes zu erhalten.

RE-H 7 Autograder – Bereitstellen einer GUI damit die Bewertungskriterien ausgewählt werden können.

Es ist eine GUI bereitgestellt, welche die eingelesene Musterlösung darstellt. Der Lehrkraft ist es möglich auszuwählen, welcher Befehl welche Wertung hat oder ob er eins-zu-eins verglichen wird. Ebenfalls wird verglichen, ob der Befehl in einem vorgegebenen Bereich liegt, oder optional in dem Sollzustand vorhanden sein soll.
